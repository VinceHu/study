# 面试速答添加指南

## 什么是面试速答

面试速答是在每道面试题开头添加的一个快速回答模板，帮助面试者快速掌握核心答案要点，无需阅读全文即可准备面试。

## 添加位置

在题目标题之后，标准答案之前添加：

```markdown
# 题目

请详细说明...

## 🎯 面试速答

::: tip
[口语化的面试回答内容]
:::

## 📝 标准答案

### 核心要点
...
```

## 格式要求

### 1. 使用 VitePress 的 tip 容器

```markdown
::: tip
内容
:::
```

**注意事项：**
- 不要添加自定义标题（如 `💬 口语化回答`）
- 不要在容器后添加 `---` 分隔线
- 直接使用 `::: tip` 开头，`::: ` 结尾

### 2. 内容要求

**风格：**
- 完全口语化，像真实面试对话一样
- 不使用代码示例
- 不使用列表、表格等格式
- 用自然的段落组织内容

**长度：**
- 支撑 2-3 分钟的完整回答
- 通常 5-8 个段落
- 每段 2-4 句话

**结构：**
1. 第一段：一句话定义 + 为什么需要
2. 中间段落：核心概念和工作原理
3. 举例说明：用文字描述具体场景
4. 注意事项：常见问题或易错点
5. 最后一段：实际应用价值

### 3. 示例模板

```markdown
## 🎯 面试速答

::: tip
[技术概念]是[简短定义]。因为[背景原因]，所以需要[该技术]来[解决什么问题]。

简单来说，[核心工作流程]是这样的：首先[步骤1]，然后[步骤2]，接着[步骤3]，这样不断循环。

这里面涉及到[几个]核心概念：[概念1]、[概念2]、[概念3]。[概念1]就是[通俗解释]。当遇到[某种情况]时，[会发生什么]。

[关键区别]很重要。[类型A]包括[例子]这些，而[类型B]主要是[例子]这些。关键点是[核心差异]。

举个例子，如果[场景描述]，那执行顺序一定是：[步骤描述]。这就是为什么[某个现象]，因为[原因解释]。

另外需要注意的是，[注意事项或常见问题]，所以[如何避免或处理]。

在实际开发中，理解[该技术]可以帮助我们[实际价值1]，也能帮助我们[实际价值2]。
:::
```

## 样式说明

面试速答区域会自动应用以下样式：
- 浅绿色背景（品牌色）
- 左侧绿色边框
- 较小的字号（0.95em）
- 适中的行高（1.8）
- 自动适配深色模式

样式已在 `.vitepress/theme/custom.css` 中定义，无需额外配置。

## 完整示例

参考 `questions/javascript/event-loop.md` 中的面试速答部分：

```markdown
## 🎯 面试速答

::: tip
事件循环是 JavaScript 处理异步操作的核心机制。因为 JavaScript 是单线程的，同一时间只能做一件事，所以需要事件循环来协调各种异步任务的执行，避免阻塞主线程。

简单来说，事件循环的工作流程是这样的：首先执行所有的同步代码，然后检查有没有微任务需要执行，如果有就把所有微任务都执行完，接着再执行一个宏任务，然后又回到检查微任务这一步，这样不断循环。

这里面涉及到三个核心概念：调用栈、宏任务队列和微任务队列。调用栈就是存放当前正在执行的代码的地方，是一个后进先出的结构。当遇到异步操作时，比如 setTimeout 或者 Promise，它们的回调函数会被放到对应的任务队列里等待执行。

宏任务和微任务的区别很重要。宏任务包括 setTimeout、setInterval、I/O 操作这些，而微任务主要是 Promise 的 then、catch、finally 回调，还有 queueMicrotask 这些。关键点是微任务的优先级比宏任务高，每次事件循环都会把所有微任务清空后，才会执行一个宏任务。

举个例子，如果代码里既有 setTimeout 又有 Promise，那执行顺序一定是：先执行同步代码，然后执行 Promise 的回调，最后才执行 setTimeout 的回调。这就是为什么即使 setTimeout 设置延迟为 0，它也不会立即执行，因为它要等微任务队列清空后才轮到它。

另外需要注意的是，浏览器的页面渲染通常发生在宏任务之间，所以如果微任务太多或者执行时间太长，也会影响页面渲染，导致页面卡顿。

在实际开发中，理解事件循环可以帮助我们更好地处理异步操作的执行顺序，避免一些意外的 bug，也能帮助我们优化性能，比如把耗时的操作分片执行，避免阻塞主线程。
:::

## 📝 标准答案
```

## 常见错误

### ❌ 错误示例 1：添加了自定义标题

```markdown
::: tip 💬 口语化回答
内容
:::
```

### ❌ 错误示例 2：添加了分隔线

```markdown
::: tip
内容
:::

---

## 📝 标准答案
```

### ❌ 错误示例 3：使用了代码块

```markdown
::: tip
事件循环的执行顺序：
```javascript
console.log('1');
setTimeout(() => console.log('2'), 0);
```
:::
```

### ❌ 错误示例 4：使用了列表格式

```markdown
::: tip
事件循环包括：
- 调用栈
- 任务队列
- 事件循环
:::
```

### ✅ 正确示例

```markdown
::: tip
事件循环是 JavaScript 处理异步操作的核心机制。它包括三个核心概念：调用栈、任务队列和事件循环本身。调用栈就是存放当前正在执行的代码的地方...
:::
```

## 更新现有题目

如需为现有题目添加面试速答：

1. 在题目标题后添加 `## 🎯 面试速答` 部分
2. 使用 `::: tip` 容器包裹内容
3. 用口语化的方式重新组织答案
4. 确保内容完整但简洁
5. 不要添加代码、列表等格式化内容

## 检查清单

添加面试速答后，请检查：

- [ ] 使用了 `## 🎯 面试速答` 标题
- [ ] 使用了 `::: tip` 容器（无自定义标题）
- [ ] 内容完全口语化，无代码示例
- [ ] 没有使用列表、表格等格式
- [ ] 没有在容器后添加 `---` 分隔线
- [ ] 内容长度适中（5-8 段）
- [ ] 覆盖了核心知识点
- [ ] 包含了实际应用价值
